## Tools

### Tool 1: search_listings

**What it does:**
Searches the loaded mock listings dataset for secondhand items that match the user's description, size, and price constraints.

**Input parameters:**
- `description` (str): Keywords describing the desired item (e.g., "vintage graphic tee").
- `size` (str): Optional size string to filter by (e.g., "M").
- `max_price` (float): Optional maximum price ceiling.

**What it returns:**
A list of dictionaries representing the matching listings. Each dictionary contains fields such as `id`, `title`, `description`, `category`, `style_tags`, `size`, `condition`, and `price`.

**What happens if it fails or returns nothing:**
If no matches are found, it returns an empty list `[]`. It does NOT raise an exception or crash.

---

### Tool 2: suggest_outfit

**What it does:**
Uses an LLM to suggest 1–2 complete outfit combinations using the newly found thrift item and the user's existing wardrobe pieces.

**Input parameters:**
- `new_item` (dict): The listing dictionary of the item the user wants to buy.
- `wardrobe` (dict): A dictionary containing an `items` list of the user's existing clothing dicts.

**What it returns:**
A non-empty string containing the LLM-generated outfit suggestions.

**What happens if it fails or returns nothing:**
If `wardrobe['items']` is empty, it does not crash. Instead, it calls the LLM with a fallback prompt to provide general styling advice for the `new_item` rather than specific wardrobe combinations.

---

### Tool 3: create_fit_card

**What it does:**
Uses an LLM to generate a short, shareable social media caption (OOTD style) about the thrifted find based on the suggested outfit.

**Input parameters:**
- `outfit` (str): The outfit suggestion string generated by Tool 2.
- `new_item` (dict): The listing dictionary for the thrifted item.

**What it returns:**
A 2–4 sentence string formatted as an authentic, casual social media caption that mentions the item, price, platform, and outfit vibe.

**What happens if it fails or returns nothing:**
If the `outfit` string is empty or missing, it returns a descriptive error message string (e.g., "Error: Missing outfit details") rather than raising a Python exception.

## Planning Loop

**How does your agent decide which tool to call next?**
After parsing the user's query, the agent calls `search_listings`. It then evaluates the return value. 
* **IF** the result is an empty list `[]`, the agent sets an error message in `session["error"]` (e.g., "No listings found") and returns the session immediately, halting execution. 
* **IF** the result contains items, it assigns the first item to `session["selected_item"]` and proceeds to call `suggest_outfit`. It saves that output, then unconditionally calls `create_fit_card`.

---

## State Management

**How does information from one tool get passed to the next?**
Information is passed using a central `session` dictionary. The top dictionary returned by `search_listings` is stored in `session["selected_item"]`. This exact dictionary is then passed as an argument into `suggest_outfit` (no re-entry required). The string returned by `suggest_outfit` is stored in `session["outfit_suggestion"]`, which is then passed directly as an argument into `create_fit_card`.

---

## Error Handling

| Tool | Failure mode | Agent response |
|------|-------------|----------------|
| search_listings | No results match the query | Sets `session['error']` to a string explaining what failed (e.g., "Couldn't find anything matching that. Try loosening your size or price constraints!") and stops the loop. |
| suggest_outfit | Wardrobe is empty | Passes the empty wardrobe gracefully to the LLM, prompting it to give general styling advice instead of relying on specific closet pieces. |
| create_fit_card | Outfit input is missing or incomplete | Returns a graceful error string like "Could not generate fit card: outfit details missing" instead of throwing a Python exception. |

## A Complete Interaction (Step by Step)

Write out what a full user interaction looks like from start to finish — tool call by tool call. Use a specific example query.

**Example user query:** "I'm looking for a vintage graphic tee under $30. I mostly wear baggy jeans and chunky sneakers. What's out there and how would I style it?"

**Step 1:**
The agent extracts the parameters and calls search_listings(description="vintage graphic tee", size=None, max_price=30.0). The tool returns a list of dictionaries. The agent takes the first dictionary (e.g., Faded Band Tee) and saves it to session["selected_item"].

**Step 2:**
The agent calls suggest_outfit(new_item=session["selected_item"], wardrobe=session["wardrobe"]). The LLM evaluates the new band tee against the user's baggy jeans and chunky sneakers, returning a text paragraph suggesting a grunge-inspired pairing. The agent saves this to session["outfit_suggestion"].

**Step 3:**
The agent calls create_fit_card(outfit=session["outfit_suggestion"], new_item=session["selected_item"]). The LLM generates a short, authentic social media caption summarizing the item and outfit. The agent saves this to session["fit_card"].

**Final output to user:**
The user sees the populated UI panels containing: the details of the Faded Band Tee, the styling paragraph recommending pairing it with their baggy jeans, and the shareable Instagram caption.
